### P0-001: 缓存模块已实现但未集成

**模块**: 故障排除助手  
**严重程度**: 🔴 严重  
**描述**: 
故障排除助手目录下已有完整的性能优化模块（cache.py、index_builder.py、preloader.py、performance_monitor.py），但query_engine.py没有使用这些模块，导致每次查询都重复计算。

**影响**:
- 查询性能低（200-500ms vs 目标<100ms）
- IDF重复计算浪费CPU
- 无法利用已有的索引优化

**复现步骤**:
1. 运行故障排除助手
2. 执行知识查询
3. 观察每次查询都耗时200-500ms

**解决方案**:
```python
# 在query_engine.py中添加
from performance import LRUCache, IndexBuilder, query_cache

class QueryEngine:
    def __init__(self, knowledge_base: List[KnowledgeItem] = None):
        self.cache = query_cache  # 使用已有的缓存实例
        self.index_builder = index_builder  # 使用已有的索引
        # ...
```

**优先级**: P0  
**预估工时**: 2小时

### P0-002: 前端知识服务无缓存机制

**模块**: Web前端  
**严重程度**: 🔴 严重  
**描述**: 
`src/services/knowledge-answer.js`中的`searchKnowledge()`函数每次执行都调用`fs.readdir()`和`fs.readFile()`扫描整个知识库目录（114个文件），导致查询耗时500ms-2s。

**影响**:
- 用户体验差，搜索响应慢
- 服务器负载高
- 网络文件系统压力大

**解决方案**:
```javascript
// 添加内存缓存层
const fileCache = new Map();
let cacheTime = 0;
const CACHE_TTL = 60000; // 1分钟缓存

async function searchKnowledge(query, options = {}) {
    // 检查缓存
    const cacheKey = `${query}:${JSON.stringify(options)}`;
    const cached = fileCache.get(cacheKey);
    if (cached && Date.now() - cacheTime < CACHE_TTL) {
        return cached;
    }
    // ... 原有逻辑
    fileCache.set(cacheKey, results);
    return results;
}
```

**优先级**: P0  
**预估工时**: 3小时  
**创建时间**: 2026-04-18

### P1-002: Vite无代码分割

**模块**: Web界面  
**严重程度**: 🟡 中等  
**描述**:

            }
        }
    }
}
