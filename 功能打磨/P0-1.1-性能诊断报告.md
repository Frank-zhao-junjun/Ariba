## 性能瓶颈分析

### 1. 故障排除助手 - 🔴 高优先级

#### 问题描述
**文件**: `Ariba实施助手/故障排除助手/core/query_engine.py`

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 无缓存机制 | 🔴 严重 | 每次查询都遍历整个知识库，重新计算TF-IDF |
| IDF重复计算 | 🔴 严重 | `_compute_idf()` 每次初始化都重新计算所有文档的IDF |
| 版本过滤低效 | 🟡 中等 | 每次查询都调用`normalize_version()`，未缓存 |
| 未使用已有优化模块 | 🔴 严重 | 已有`cache.py`和`index_builder.py`但未集成 |

#### 代码问题示例
```python
# 问题1: 每次search都遍历所有文档
for item in self.knowledge_base:  # O(n) 复杂度
    # 每次都计算TF-IDF
    title_score = self._calculate_tfidf(item.title, keywords)

# 问题2: IDF未缓存且重复计算
def _compute_idf(self):
    for item in self.knowledge_base:  # 重复遍历
        words = set(self._tokenize(item.get_searchable_text()))
```

#### 性能影响估算
- **当前单次查询耗时**: 200-500ms（取决于知识库大小）
- **目标耗时**: <100ms
- **差距**: 2-5倍

### 2. 清单生成器 - 🟡 中等优先级

#### 问题描述
**文件**: `Ariba实施助手/实施检查清单生成器/core/checklist_generator.py`

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 模板硬编码 | 🟡 中等 | 所有清单模板直接存储在内存字典中，无懒加载 |
| 无缓存机制 | 🟡 中等 | 每次实例化都调用`_load_builtin_templates()` |
| 筛选逻辑低效 | 🟢 轻微 | 版本和模块筛选使用线性扫描 |

#### 代码问题示例
```python
# 问题: 每次初始化都加载所有模板
def _load_builtin_templates(self) -> Dict[str, Any]:
    return {
        "requirements_analysis": { ... },  # 大量内联数据
        "system_configuration": { ... },
        # ...
    }

# 问题: 无缓存的版本规范化
def normalize_version(version: str) -> str:
    if not version.startswith("#"):
        version = "#" + version
    return version
```

#### 性能影响估算
- **当前初始化耗时**: 50-100ms
- **目标耗时**: <200ms（加载）
- **状态**: 勉强达标，但优化空间大

### 3. Web前端知识服务 - 🔴 高优先级

#### 问题描述
**文件**: `src/services/knowledge-answer.js`

| 问题 | 严重程度 | 说明 |

| 每次查询扫描全目录 | 🔴 严重 | `searchKnowledge()` 每次都调用`fs.readdir()`和`fs.readFile()` |

#### 代码问题示例
```javascript
// 问题: 每次搜索都扫描整个目录
async function searchKnowledge(query, options = {}) {
    await scanDirectory(kbPath);  // 每次都执行
    // ...
}

// 问题: 重复创建正则表达式
const regex = new RegExp(keyword, 'gi');  // 每个关键词都创建新正则
```

#### 性能影响估算
- **当前单次查询耗时**: 500ms-2s（网络文件系统）
- **目标耗时**: <100ms
- **差距**: 5-20倍

## 诊断结论

### 主要瓶颈
1. 🔴 **故障排除助手**: 虽有优化代码但未集成，导致每次查询重复计算

3. 🟡 **清单生成器**: 模板硬编码，无懒加载

### 优化潜力
- 集成已有缓存模块可立即提升 **80-90%** 性能
- 添加前端缓存可进一步提升 **90%** 性能
- 总体优化后可达标目标性能

### 建议优先级
1. 集成`cache.py`和`index_builder.py`到`query_engine.py`
2. 为前端`knowledge-answer.js`添加Node.js内存缓存
3. 优化Vite构建配置
4. 清单生成器懒加载优化
