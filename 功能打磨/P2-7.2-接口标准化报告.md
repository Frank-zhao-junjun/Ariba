## 1. 接口现状分析

### 1.1 发现的问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 接口格式不统一 | 各模块使用不同的请求/响应格式 | 难以统一处理 |
| 缺乏版本管理 | 无API版本控制 | 升级困难 |
| 错误码不一致 | 错误处理方式各异 | 调试困难 |
| 缺乏文档 | 接口无标准文档 | 集成成本高 |

### 1.2 现有接口清单

| 模块 | 接口类型 | 问题 |
|------|----------|------|
| QueryEngine | 直接调用 | 无标准化 |
| ChecklistGenerator | 直接调用 | 无标准化 |
| Web API | RESTful | 无版本控制 |
| 缓存模块 | 直接调用 | 无标准化 |

## 2. 标准化方案

### 2.1 统一请求/响应格式

```python
# 统一请求格式
@dataclass
class APIRequest:
    action: str                    # 操作类型
    params: Dict[str, Any]         # 参数
    request_id: Optional[str]       # 请求ID
    timestamp: float                # 时间戳
    version: str = "1.0"            # 接口版本

# 统一响应格式
@dataclass
class APIResponse:
    success: bool                   # 是否成功
    data: Any                       # 响应数据
    error: Optional[str]           # 错误信息
    error_code: Optional[str]       # 错误码
    request_id: Optional[str]       # 关联请求ID
    timestamp: float                # 响应时间戳
```

### 2.2 标准接口定义

#### 查询接口 (IQueryEngine)
```python
class IQueryEngine(ABC):
    def search(query, version, limit) -> List[Dict]
    def query_by_symptom(symptom) -> List[Dict]
    def get_related_knowledge(knowledge_id) -> Optional[Dict]
```

#### 清单生成接口 (IChecklistGenerator)
```python
class IChecklistGenerator(ABC):
    def generate(phase, module, version, options) -> List[Dict]
    def get_phases() -> List[str]
    def get_modules() -> List[str]
```

### 2.3 错误码规范

| 错误码 | 含义 | HTTP状态码 |
|--------|------|-----------|
| ROUTE_NOT_FOUND | 路由不存在 | 404 |
| VALIDATION_ERROR | 参数验证失败 | 400 |
| RATE_LIMIT | 速率限制 | 429 |
| INTERNAL_ERROR | 内部错误 | 500 |
| UNAUTHORIZED | 未授权 | 401 |
| FORBIDDEN | 禁止访问 | 403 |

## 3. 实现内容

### 3.1 创建的文件

| 文件 | 用途 |
|------|------|
| `specifications.py` | 接口规范定义 |
| `api_gateway.py` | API网关实现 |

### 3.2 核心组件

#### APIGateway - API网关
```python
gateway = APIGateway()
gateway.register_endpoint(endpoint)
gateway.handle_request(method, path, params)
```

#### RequestValidator - 请求验证器
```python
validator = RequestValidator()
valid, error = validator.validate_request(request, schema)
```

#### RouteConfig - 路由配置
```python
route = RouteConfig(
    path="/search",
    method=HTTPMethod.POST,
    handler=search_handler,
    description="搜索接口",
    params_schema={...},
    auth_required=True,
    rate_limit=100
)
```

### 3.3 装饰器支持

```python
@route("/search", HTTPMethod.POST, "搜索故障知识")
@validate({"type": "object", "required": ["query"]})
def search(request: APIRequest):
    return adapter.search(request.params["query"])
```

## 7. 测试验证

### 7.1 接口一致性测试
```python
def test_api_consistency():
    gateway = APIGateway()
    # 测试所有路由返回统一格式
    for route in gateway.get_route_info():
        response = gateway.handle_request(
            route.method, route.path, {}
        )
        assert isinstance(response, APIResponse)
        assert hasattr(response, 'success')
        assert hasattr(response, 'error_code')
```

### 7.2 验证规则测试
```python
def test_validation():
    schema = {
        "required": ["query"],
        "properties": {
            "query": {"type": "string"},
            "limit": {"type": "integer"}
        }
    }
    request = APIRequest(action="test", params={"query": 123})
    valid, error = RequestValidator.validate_request(request, schema)
    assert not valid
    assert "string" in error
```
