## 1. 配置现状分析

### 1.1 发现的问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 配置分散 | 各模块独立配置，无统一管理 | 维护困难 |
| 硬编码值 | 部分配置硬编码在代码中 | 灵活性差 |
| 缺乏验证 | 无配置验证机制 | 运行时错误 |
| 无热更新 | 配置变更需重启 | 影响可用性 |
| 环境差异 | 无环境适配机制 | 部署复杂 |

### 1.2 现有配置清单

| 模块 | 配置项 | 问题 |
|------|--------|------|
| QueryEngine | cache_ttl, max_results | 硬编码 |
| ChecklistGenerator | template_dir | 本地路径 |
| Web API | port, debug | 分散 |
| 日志 | log_level | 无统一管理 |

## 2. 优化方案

### 2.1 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    配置管理层                            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐       │
│  │ConfigManager│  │ConfigLoader │  │  Validator  │       │
│  └─────────────┘  └─────────────┘  └─────────────┘       │
└─────────────────────────────────────────────────────────┘
                          │
┌─────────────────────────────────────────────────────────┐
│                    配置源层                              │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │
│  │ DEFAULT │  │   ENV   │  │  FILE   │  │ RUNTIME │    │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘     │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心功能

1. **集中管理**: 统一的配置管理器
2. **多源加载**: 支持文件、环境变量、默认值
3. **Schema验证**: 类型检查、范围验证、选项验证
4. **热更新**: 配置文件监视、变更通知
5. **环境适配**: 开发/测试/生产环境自动切换

## 3. 实现内容

### 3.1 创建的文件

| 文件 | 用途 |
|------|------|
| `shared/config/__init__.py` | 配置管理入口 |
| `shared/config/loaders.py` | 配置加载器 |
| `shared/config/validation.py` | 验证与热更新 |
| `shared/config/examples.py` | 使用示例 |

### 3.2 核心组件

#### ConfigManager - 配置管理器
```python
# 单例模式，线程安全
cm = get_config_manager()

# 多源加载
cm.load_defaults({...})
cm.load_from_file("config.yaml")
cm.load_from_env()

# 获取/设置
cm.get("key", default)
cm.set("key", value)

# Schema验证
cm.add_schema("app", schema)
cm.validate()
```

#### ConfigLoader - 配置加载器
```python
loader = ConfigLoader()
loader.search_paths = ["./config", "/etc/app"]
loader.env_prefix = "ARIBAS_"
config = loader.load("app")
```

#### ConfigHotReloader - 热更新器
```python
reloader = ConfigHotReloader(cm)
reloader.watch_file("config/app.yaml", callback=on_change)
reloader.start()
```

#### EnvironmentAdapter - 环境适配
```python
env = EnvironmentAdapter.get_environment()
config = EnvironmentAdapter.get_env_config()
```

### 3.3 配置Schema

```python
schema = ConfigSchema()
schema.add_field(
    "cache_ttl", int,
    default=3600,
    description="缓存TTL(秒)",
    validator=lambda v: 0 < v <= 86400
)
schema.add_field(
    "log_level", str,
    choices=["DEBUG", "INFO", "WARNING", "ERROR"]
)
```

## 4. 配置格式

### 4.1 YAML格式

```yaml
# config/app.yaml
database:
  host: localhost
  port: 5432
  name: ariba_db

cache:
  enabled: true
  ttl: 3600

troubleshooting:
  max_results: 10
  similarity_threshold: 0.7

checklist:
  template_dir: templates
  cache_enabled: true
```

### 4.2 JSON格式

```json
{
  "database": {
    "host": "localhost",
    "port": 5432,
    "name": "ariba_db"
  },
  "cache": {
    "enabled": true,
    "ttl": 3600
  }
}
```

### 4.3 环境变量

```bash
# 以ARIBAS_为前缀
export ARIBAS_DATABASE__HOST=production-db
export ARIBAS_DATABASE__PORT=5432
export ARIBAS_CACHE__ENABLED=true
```

## 5. 验证规则

### 5.1 预定义验证

| 配置项 | 类型 | 范围/选项 |
|--------|------|-----------|
| cache_ttl | int | 0 < x <= 86400 |
| max_results | int | 0 < x <= 100 |
| similarity_threshold | float | 0 <= x <= 1 |
| log_level | str | DEBUG, INFO, WARNING, ERROR |
| environment | str | development, testing, staging, production |

### 5.2 自定义验证

```python
validator = ConfigValidator(ValidationLevel.STRICT)
validator.add_rule("custom_field", lambda v: v > 0)
```

### 6.1 文件监视

```python
reloader.watch_file("config/app.yaml")
reloader.add_callback(lambda path, data: print(f"Changed: {path}"))

```

```python
detector = ConfigChangeDetector()

changes = detector.detect_changes("app", new_config)
```

### 6.3 回滚机制

```python
history = detector.get_history("app", limit=10)

```

## 9. 使用建议

### 9.1 目录结构

```
config/
├── default.yaml       # 默认配置
├── development.yaml   # 开发环境
├── testing.yaml       # 测试环境
├── staging.yaml       # 预发布环境
└── production.yaml    # 生产环境
```

### 9.2 命名规范

```bash
# 使用双下划线分隔嵌套
ARIBAS_DATABASE__HOST=localhost
ARIBAS_CACHE__TTL=3600
```

### 9.3 最佳实践

1. **始终提供默认值**: 避免配置缺失导致的错误
2. **使用Schema验证**: 提前发现配置问题
3. **环境变量优先**: 敏感配置使用环境变量
4. **定期备份**: 重要配置变更前备份
5. **监控变更**: 使用热更新监听配置变更
