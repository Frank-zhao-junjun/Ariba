### DEF-TC3.3：推荐引擎参数类型不匹配

#### 问题描述
```
RecommendationEngine.recommend_checklist_items() got an unexpected keyword argument 'profile'
```

#### 问题根源
- 测试代码使用 `profile=` 和 `existing_items=` 关键字参数调用方法
- 但 `recommend_checklist_items()` 方法定义的是 `project_profile` 和 `base_checklist` 参数

#### 修复方案
修改 `recommend_checklist_items` 方法，支持新参数名并保持向后兼容：

```python
def recommend_checklist_items(
    self,
    profile: Dict[str, Any] = None,        # 新参数名
    existing_items: List[Dict[str, Any]] = None,  # 新参数名
    # 兼容旧参数名
    project_profile: Dict[str, Any] = None,
    base_checklist: List[Dict[str, Any]] = None
) -> Dict[str, Any]:
    # 参数兼容处理
    if profile is None and project_profile is not None:
        profile = project_profile
    if existing_items is None and base_checklist is not None:
        existing_items = base_checklist
```

#### 修复文件
- `Ariba实施助手/实施检查清单生成器/recommendation/recommendation_engine.py`

#### 验证结果
- ✅ 新参数名调用 `profile=, existing_items=` → PASS
- ✅ 旧参数名调用 `project_profile=, base_checklist=` → PASS
- ✅ 位置参数调用 → PASS
- ✅ 空参数防御 → PASS

### DEF-TC6.2：版本标签格式差异

#### 问题描述
- 故障排除助手使用 `#V2605` 格式（带#前缀）
- 清单生成器使用 `V2605` 格式（无#前缀）
- 导致版本过滤和一致性验证失败

#### 问题根源
1. 清单生成器的 `SUPPORTED_VERSIONS` 定义为 `["V2505", "V2602", ...]`
2. 模板中的版本标签为 `["V2505", "V2602", ...]`
3. 知识库使用 `#V2505` 格式

#### 修复方案

**1. 创建共享版本工具**
创建统一版本处理模块 `Ariba实施助手/shared/version_utils.py`：

```python
SUPPORTED_VERSIONS = {"#V2505", "#V2602", "#V2604", "#V2605", "#VNextGen", "#VClassic"}

def normalize_version(version: str) -> str:
    """规范化版本标签为标准格式 #VXXXX"""
    version = version.strip().upper()
    if not version.startswith("#"):
        version = "#" + version
    return version
```

**2. 更新清单生成器**
- 更新 `SUPPORTED_VERSIONS` 使用 `#VXXXX` 格式
- 更新所有模板版本标签为 `#VXXXX` 格式
- 添加版本规范化处理，支持新旧格式输入

**3. 保留版本字段**
在生成的清单项中保留 `versions` 字段用于一致性验证：

```python
items.append({
    ...
    "versions": item["versions"],  # 保留版本标签
    ...
})
```

#### 修复文件
- `Ariba实施助手/shared/version_utils.py`（新增）
- `Ariba实施助手/实施检查清单生成器/core/checklist_generator.py`

#### 验证结果
- ✅ 共享版本工具存在 → PASS
- ✅ 规范化 `V2605` → `#V2605` → PASS
- ✅ 规范化 `#V2605` → `#V2605` → PASS
- ✅ 清单生成器版本格式统一 → PASS
- ✅ 版本过滤格式兼容 → PASS
