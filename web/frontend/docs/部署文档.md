## 快速部署

### 一键部署脚本

```bash
#!/bin/bash
# deploy.sh - 快速部署脚本

set -e

echo "=== SAP Ariba 实施助手部署脚本 ==="

# 1. 克隆代码
echo "[1/5] 克隆代码..."
git clone https://github.com/your-org/ariba-assistant.git
cd ariba-assistant

# 2. 安装前端依赖
echo "[2/5] 安装前端依赖..."
cd web/frontend
npm install

# 3. 构建前端
echo "[3/5] 构建前端..."
npm run build

# 4. 安装后端依赖
echo "[4/5] 安装后端依赖..."
cd ../server
pip install -r requirements.txt

# 5. 启动服务
echo "[5/5] 启动服务..."
# 开发环境
uvicorn main:app --reload --host 0.0.0.0 --port 8000 &

echo "部署完成！访问 http://localhost:8000"
```

### 部署时间估算

| 步骤 | 耗时 |
|-----|------|
| 克隆代码 | 1-5 分钟 |
| 安装依赖 | 5-15 分钟 |
| 构建前端 | 2-5 分钟 |
| 配置服务 | 5-10 分钟 |
| **总计** | **15-35 分钟** |

## Docker 部署

### 前端 Dockerfile

```dockerfile
# frontend/Dockerfile
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

# 生产镜像
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 后端 Dockerfile

```dockerfile
# server/Dockerfile
FROM python:3.10-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### docker-compose.yml

```yaml
version: '3.8'

services:
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - app-network

  backend:
    build:
      context: ./server
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=sqlite:///./data/app.db
      - CORS_ORIGINS=https://example.com
    volumes:
      - ./data:/app/data
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
```

### 构建和运行

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 反向代理配置

### Nginx 配置

```nginx
# /etc/nginx/conf.d/ariba-assistant.conf

upstream backend {
    server 127.0.0.1:8000;
    keepalive 32;
}

server {
    listen 80;
    server_name ariba.example.com;
    
    # 重定向到 HTTPS（生产环境）
    # return 301 https://$server_name$request_uri;
    
    # 前端静态文件
    root /var/www/ariba-assistant/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}
```

### Apache 配置

```apache
# .htaccess 或 httpd.conf
<VirtualHost *:80>
    ServerName ariba.example.com
    DocumentRoot /var/www/ariba-assistant/dist
    
    # 前端路由
    <Directory /var/www/ariba-assistant/dist>
        RewriteEngine On
        RewriteBase /
        RewriteRule ^index\.html$ - [L]
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule . /index.html [L]
    </Directory>
    
    # API 代理
    ProxyPreserveHost On
    ProxyPass /api/ http://127.0.0.1:8000/api/
    ProxyPassReverse /api/ http://127.0.0.1:8000/api/
</VirtualHost>
```

## 运维监控

### 日志配置

```python
# logging_config.py
import logging
from logging.handlers import RotatingFileHandler
import sys

def setup_logging():
    logger = logging.getLogger()
    logger.setLevel(logging.INFO)
    
    # 文件日志
    file_handler = RotatingFileHandler(
        'logs/app.log',
        maxBytes=10*1024*1024,  # 10MB
        backupCount=5
    )
    file_handler.setFormatter(logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    ))
    logger.addHandler(file_handler)
    
    # 控制台日志
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setFormatter(logging.Formatter(
        '%(asctime)s - %(levelname)s - %(message)s'
    ))
    logger.addHandler(console_handler)
    
    return logger
```

### 健康检查

```bash
# HTTP 健康检查
curl -f http://localhost:8000/api/dashboard/health

# 进程检查
ps aux | grep uvicorn

# 端口检查
netstat -tlnp | grep 8000
```

### 监控脚本

```bash
#!/bin/bash
# monitor.sh - 监控脚本

# 检查进程
if ! pgrep -f "uvicorn main:app" > /dev/null; then
    echo "[ALERT] Backend process not running!"
    # 自动重启
    cd /app && uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4 >> app.log 2>&1 &
fi

# 检查 API
if ! curl -sf http://localhost:8000/api/dashboard/health > /dev/null; then
    echo "[ALERT] API not responding!"
fi

echo "[OK] System healthy"
```

### 定时任务

```bash
# crontab -e
# 每5分钟检查一次
*/5 * * * * /app/monitor.sh >> /var/log/monitor.log 2>&1

# 每天凌晨清理日志
0 0 * * * find /app/logs -name "*.log" -mtime +7 -delete
```

## 故障排查

### 常见问题

#### 1. 前端构建失败

```bash
# 清理缓存
rm -rf node_modules package-lock.json
npm cache clean --force
npm install

# 检查 Node 版本
node -v  # 需要 >= 18.x
```

#### 2. 后端启动失败

```bash
# 检查端口占用
lsof -i :8000

# 检查 Python 版本
python --version  # 需要 >= 3.10

# 检查依赖
pip install -r requirements.txt --force-reinstall
```

#### 3. API 请求超时

```nginx
# 增加 Nginx 超时时间
proxy_connect_timeout 300s;
proxy_send_timeout 300s;
proxy_read_timeout 300s;
```

#### 4. CORS 跨域问题

```python
# 后端 CORS 配置
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://example.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
