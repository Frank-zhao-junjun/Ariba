# Ariba实施助手 Web界面部署指南

## 环境要求

- **Node.js**: >= 18.0.0
- **Python**: >= 3.9
- **npm** 或 **pnpm**

## 项目结构

```
Ariba实施助手/web/
├── frontend/          # 前端项目 (React + TypeScript + Vite)
├── backend/            # 后端项目 (FastAPI)
├── docs/               # 文档
└── .ralph/             # Ralph开发日志
```

## 开发环境部署

### 1. 后端部署

```bash
cd Ariba实施助手/web/backend

# 安装依赖
pip install -r requirements.txt

# 启动开发服务器
python api_server.py
# 或使用uvicorn
uvicorn api_server:app --reload --port 8000
```

服务启动后，访问：
- API文档: http://localhost:8000/docs
- ReDoc文档: http://localhost:8000/redoc
- 健康检查: http://localhost:8000/health

### 2. 前端部署

```bash
cd Ariba实施助手/web/frontend

# 安装依赖
npm install
# 或
pnpm install

# 启动开发服务器
npm run dev
```

服务启动后，访问：http://localhost:3000

### 3. 代理配置

Vite已配置API代理，将 `/api` 请求转发到 `http://localhost:8000`。

## 生产环境部署

### 后端部署

```bash
cd Ariba实施助手/web/backend

# 安装生产依赖
pip install -r requirements.txt

# 使用gunicorn部署
pip install gunicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker api_server:app -b 0.0.0.0:8000
```

### 前端构建

```bash
cd Ariba实施助手/web/frontend

# 构建生产版本
npm run build

# 构建产物位于 dist/ 目录
```

### Nginx配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 前端静态文件
    location / {
        root /path/to/Ariba实施助手/web/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
    
    # API代理
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## Docker部署

### Dockerfile (后端)

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
EXPOSE 8000
CMD ["uvicorn", "api_server:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Docker Compose

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
  frontend:
    build: ./frontend
    ports:
      - "3000:80"
```

## 环境变量

### 后端环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| HOST | 0.0.0.0 | 监听地址 |
| PORT | 8000 | 监听端口 |
| DEBUG | false | 调试模式 |

### 前端环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| VITE_API_BASE_URL | /api | API基础URL |

## 验证部署

部署完成后，访问以下地址验证：

1. 前端: http://localhost:3000
2. 后端API: http://localhost:8000
3. API文档: http://localhost:8000/docs
4. 健康检查: http://localhost:8000/health

## 故障排查

### 前端无法连接后端

1. 检查后端服务是否运行: `curl http://localhost:8000/health`
2. 检查Vite代理配置: `vite.config.ts`
3. 检查CORS配置: `backend/api_server.py`

### 构建失败

```bash
# 清理缓存
rm -rf node_modules package-lock.json
npm install
```
