# Ralph开发决策记录 - US1

> 记录所有技术选型和设计决策

## 2026-04-17 设计决策

### D1: 数据结构选择
**决策**：采用Pydantic模型定义知识条目
**理由**：
- 类型安全，IDE自动补全
- 便于JSON序列化/反序列化
- 支持字段验证
**备选**：dataclass / TypedDict / 普通dict
**结论**：选择Pydantic BaseModel

### D2: 版本标签格式
**决策**：使用前缀`V`+版本号格式
**格式**：`#V2505, #V2602, #V2604, #V2605, #VNextGen, #VClassic`
**理由**：与现有知识库标签格式保持一致

### D3: 查询匹配算法
**决策**：基于关键词匹配度 + TF-IDF加权
**理由**：实现简单、效果好、资源占用低
**备选**：BM25 / 语义向量检索
**结论**：MVP阶段使用关键词匹配，TF-IDF加权排序

### D4: 版本过滤策略
**决策**：多版本支持 OR 匹配
**逻辑**：包含任一指定版本标签的知识条目通过过滤
**理由**：一条知识可能同时适用于多个版本

### D5: 项目结构
**决策**：模块化分层结构
```
core/      - 核心查询引擎
models/    - 数据模型
utils/     - 工具函数
tests/     - 测试用例
fixtures/  - 测试数据
docs/      - 文档
```
**理由**：清晰分离关注点，便于后续扩展
