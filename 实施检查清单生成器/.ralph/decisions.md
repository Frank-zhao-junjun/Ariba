# Ralph决策记录 - 实施检查清单生成器

## 技术选型决策

### 清单数据存储格式
- **决策**: JSON文件存储 + 内存缓存
- **原因**: MVP阶段避免引入数据库依赖，便于版本控制和调试
- **日期**: 2026-04-17

### 版本适配策略
- **决策**: 采用版本标签过滤机制
- **版本列表**: V2505, V2602, V2604, V2605, VNextGen, VClassic
- **原因**: 不同版本功能存在差异，通过标签实现兼容
- **日期**: 2026-04-17

### 输出格式策略
- **决策**: 支持Markdown、JSON、Excel导出
- **原因**: Markdown便于查看，JSON便于程序处理，Excel便于协作
- **日期**: 2026-04-17

### 智能推荐算法
- **决策**: 基于规则的推荐 + 加权评分
- **原因**: MVP阶段避免复杂ML模型，基于规则的推荐更可控、可解释
- **日期**: 2026-04-17

### 项目结构决策
- **决策**: 按功能模块划分目录
- **目录**: core, recommendation, template, tracking, analytics
- **原因**: 清晰的职责分离，便于维护和扩展
- **日期**: 2026-04-17
