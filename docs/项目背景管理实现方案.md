# 项目背景管理功能实现方案

## 一、需求概述

项目背景管理是所有项目级功能的基础，为需求分析、方案设计、蓝图设计等功能提供项目上下文信息。

### 核心功能
1. **新建项目**：问卷引导填写、附件上传并解析、两者结合模式
2. **项目背景管理**：基本信息、Ariba信息、IT环境、业务背景、团队信息
3. **项目管理**：列表查看、项目切换、编辑、归档
4. **项目背景调用**：项目级功能自动加载，基于背景定制输出

---

## 二、技术方案

### 2.1 整体架构决策

#### 前端界面：需要实现
- **决策**：需要前端页面
- **理由**：
  - 项目背景信息复杂（5大模块，30+字段）
  - 问卷引导交互需要可视化界面
  - 文件上传需要UI支持
  - 项目管理（列表、切换、编辑）需要直观操作
- **实现方式**：复用现有网站风格，新增独立页面

#### 数据存储方案：JSON文件
- **决策**：使用JSON文件存储
- **理由**：
  - 项目已采用JSON文件存储（knowledge目录、data目录）
  - 无需额外数据库依赖，简化部署
  - 数据量可控（单用户场景）
  - 便于版本控制和迁移
- **存储位置**：`src/data/projects/`

#### API接口设计：REST API
- **决策**：提供REST API接口
- **理由**：
  - 与现有架构一致（已有/api/requirement等接口）
  - 支持前端AJAX调用
  - 便于后续扩展为Agent调用

### 2.2 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生HTML/CSS/JavaScript | 与现有页面保持一致 |
| 后端 | Express.js路由 | 新增/api/projects路由 |
| 存储 | JSON文件 | fs模块读写 |
| 文件解析 | markitdown | 解析PDF/Word等附件 |
| 问卷引导 | 分步表单 | JavaScript动态渲染 |

4. LLM提取关键信息（目标、痛点）

6. 用户可手动调整

### 6.2 附件解析实现

```javascript
// 附件上传处理
router.post('/:id/upload', upload.single('file'), async (req, res) => {
  const { id } = req.params;
  const file = req.file;

  // 使用markitdown解析
  const markitdown = require('markitdown');
  const md = new MarkItDown();
  const result = md.convert(file.path);

  // LLM提取关键信息
  const extractedInfo = await extractKeyInfo(result.text_content);

  // 更新项目附件信息
  const project = loadProject(id);
  project.attachments.push({
    filename: file.originalname,
    uploadTime: new Date().toISOString(),
    parsedInfo: extractedInfo
  });

  saveProject(id, project);
  res.json({ success: true, filename: file.originalname, parsedInfo: extractedInfo });
});
```

## 九、技术风险与应对

| 风险 | 应对措施 |

## 十、总结

### 核心优势
1. **低依赖**：无需额外数据库，简化部署
2. **可扩展**：配置化问卷，易于添加新字段
3. **用户友好**：分步引导，降低填写门槛
4. **AI增强**：附件解析 + LLM提取，提升效率
