## 一、评估方法说明

### 四步评估流程

```
第1步: GSTACK CEO评估    → 战略契合度判断
第2步: RICE+KANO评分     → 量化指标计算
第3步: Ralph多角色评估   → 团队共识验证
第4步: 综合决策          → 最终优先级排序
```

### RICE评分公式

```
RICE = (Reach × Impact × Confidence) / Effort

- Reach (覆盖度): 每个季度受影响用户数 (满分100)
- Impact (影响力): 对用户的影响程度 (0.25/0.5/1/2/3)
- Confidence (置信度): 评估的确定性 (50%/80%/100%)
- Effort (工作量): 需要的人月数
```

### KANO分类与系数

| KANO类型 | 描述 | 系数 | 判断标准 |
|---------|------|------|---------|
| M (Must-have) | 基本型需求，必备功能 | 2.0 | 有则满意，无则强烈不满 |
| P (Performance) | 期望型需求，越多越好 | 1.5 | 有则更满意，无则不满 |
| A (Attractive) | 魅力型需求，惊喜功能 | 1.2 | 有则惊喜，无则无所谓 |
| I (Indifferent) | 无差异需求，有无都行 | 0.5 | 有无都不影响满意 |
| R (Reverse) | 反向需求，做了反而差 | 0 | 做了反而被嫌弃 |

### 综合评分权重

```
总分 = 第2步(RICE×KANO) × 40% + 第3步(多角色平均) × 60%
```

### 决策标准

| 分值区间 | 决策 | 颜色标识 |
|---------|------|---------|
| 8-10分 | 🟢 立即开发 | 优先级最高 |
| 6-7.9分 | 🟡 排队等待 | 下个迭代 |
| 4-5.9分 | 🟠 暂缓评估 | 储备功能 |
| 0-3.9分 | 🔴 拒绝/搁置 | 暂不考虑 |
